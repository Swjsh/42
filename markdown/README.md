# markdown/ — consolidated documentation

All human-authored docs live here, organized by topic. Operational files
(`automation/`, `.claude/`, `journal/`, `strategy/candidates/`, `analysis/`)
stay where the engine reads them.

| Folder | Contents |
|---|---|
| `0dte/` | SPY 0DTE strategy, playbook, patterns, key levels, J-edge, risk rules |
| `futures/` | MNQ/MES contract specs, margin, sessions, sources |
| `research/` | Backtest findings, studies, swarm benchmarks, R&D writeups |
| `planning/` | Roadmaps, checklists, gameplans, daily-review process |
| `doctrine/` | Lessons learned, doctrine archive, edge doctrine |
| `specs/` | Engine design + wiring specs |
| `audits/` | Point-in-time health checks, postmortems, status snapshots |
| `infra/` | Setup, MCP install, kitchen spec, skills catalog, tooling |
