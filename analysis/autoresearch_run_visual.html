Smart watchdog — what it now catches

Detection rules (built tonight)

  1. 0 KEEPs in last batch across all modes → flag "stuck"
  2. 0 KEEPs across N batches → flag "STUCK; review hard gates"
  3. ≥2 REVERTs with val P&L > $200 + sharpe > 1.0 → flag "gates may be too strict" (this is the exact gate-bug signature)
  4. Param tried > 4× with 0 KEEPs → flag as dead-end (proposer should bias away)
