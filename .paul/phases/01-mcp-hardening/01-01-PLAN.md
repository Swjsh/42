<objective>
## Goal
Wire the pre-built ribbon fallback compute core into both heartbeats so that a TV error or stale
chart automatically falls back to Alpaca bars instead of emitting SKIP. Also make the watchdog
force-restart TV when CDP is alive but the heartbeat has been frozen for >6 min — catching the
hung-MCP-bridge case the port check misses.

## Purpose
The 2026-06-24 morning miss: TV hung → heartbeat blind → PMH-rejection scalp missed. The ribbon
fallback compute core (ribbon_fallback.py, 11/11 tests, Layer-1a) has been sitting unused. This
plan wires it into production. The watchdog change closes the second gap: CDP alive ≠ bridge alive.

## Output
- `automation/scripts/ribbon_cli.py` — CLI wrapper for ribbon_fallback.compute_ribbon()
- Both heartbeat.md files — TV fallback branch added after `data_get_ohlcv` call
- `setup/scripts/run-tv-watchdog.ps1` — hung-bridge detection + force restart
</objective>

## AC-2: Heartbeat falls back to Alpaca bars on TV failure
```gherkin
Given the heartbeat.md Step 3 TV data read (data_get_ohlcv) returns an error OR bars are stale
When the heartbeat tick runs
Then it calls mcp__alpaca__get_stock_bars(symbol="SPY", timeframe="5Min", limit=60)
And runs python automation/scripts/ribbon_cli.py with the close prices
And uses the returned ribbon values for all downstream ribbon checks
And logs TV_FALLBACK_ACTIVE in the one-line output and decisions.jsonl
And only emits SKIP_TV_DATA_STALE if the Alpaca bars call also fails or returns <2 bars
```

## AC-3: Watchdog force-restarts TV on hung bridge
```gherkin
Given CDP port 9222 is alive (returns 200 on /json/version)
And Gamma_Heartbeat LastRunTime is more than 6 minutes ago during RTH (09:35-15:55 ET)
When the watchdog fires
Then it kills the TradingView process and relaunches via launch_tv_debug.ps1 -Kill
And logs tv_action="relaunch_hung_bridge" to tv-watchdog-status.json and tv-watchdog.jsonl
```
