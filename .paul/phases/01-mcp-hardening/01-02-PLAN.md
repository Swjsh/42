<objective>
## Goal
Add explicit 3-retry exponential-backoff instructions to both heartbeat prompts for critical Alpaca
MCP calls — specifically `place_option_order`, `get_account_info`, and `get_all_positions`. A single
429/503/network error currently drops silently; GhostOrderReconciler catches the miss after the
fact. The retry layer makes heartbeat self-healing for transient Alpaca errors.

## Purpose
2026-05-18 incident: heartbeat timed out during bracket placement, leaving a parent order
unprotected (no TP1/stop legs). Single-call failure = silent order miss; GhostReconciler is
detection, not prevention. This adds prevention.

## Output
Both heartbeat.md files with Alpaca retry instructions for critical calls.
</objective>
