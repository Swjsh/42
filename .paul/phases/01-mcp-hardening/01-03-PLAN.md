<objective>
## Goal
Two cheap operational hardening changes that require no code logic:
1. Promote `Gamma_McpWeeklyAudit` (Sunday-only) to `Gamma_McpDailyAudit` (every day 18:30 ET),
   closing the 5-day RTH blind gap where a hung TV or dead Alpaca bridge goes undetected Mon–Sat.
2. Stagger `Gamma_Heartbeat_Aggressive` start time from 09:30 to 09:31 ET, breaking the correlated
   rate-limit risk where both accounts hammer the Alpaca API simultaneously on every 3-min boundary.

## Purpose
Current weekly audit leaves 5 business days where a broken MCP bridge is invisible until Sunday.
Current heartbeat stagger = 0s, so a rate-limit event hits both accounts at the same instant.

## Output
- `setup/install-mcp-daily-audit.ps1` — installs the new daily task, unregisters the old weekly one
- `setup/scripts/run-mcp-daily-audit.ps1` — thin wrapper that calls the existing mcp-audit prompt
- `automation/state/SCHEDULED-TASKS.md` — updated task registry (McpWeeklyAudit retired, McpDailyAudit added)
- Bold heartbeat task re-registered at 09:31 ET (via PowerShell command, documented in plan for J to run)
</objective>
